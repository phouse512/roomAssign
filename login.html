RoomAssign Prototype

Welcome to the first prototype of RoomAssign!

This is what this prototype is built to do:
  • display what RoomAssign is fundamentally supposed to do: allow for dorm research, roommate finding, and housing application - no login is needed
  • use HTML5 local storage to allow for a simulation of saved and finished housing applications
this prototype is NOT built to:
  • offer legitimate and accurate information about how housing information at Northwestern
  • have a real backend that allows for different users and correctly stored data
Other Notes:
Throughout this application, there are only three actual hardcoded rooms available to apply for - 205 (single), 310 (double), 410 (triple)

As mentioned before, data is stored using HTML5's localStorage which uses clients' local browsers. If this is your first time testing the prototype, clear the cache below so that there is no previous saved or finished applications that will take away from your ability to test the entire protoype.
